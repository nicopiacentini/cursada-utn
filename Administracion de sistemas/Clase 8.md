### Diferencia entre Product manager, Project manajer, Product owner y scrum master
#### Project manager
Existe tanto en predictivo como en metodologias agiles. Se encarga de elegir la metodologia correcta para un determinado proyecto.
Es la persona encargada de la direccion y gestion de proyectos. Define, planifica y ejecuta un proyecto. Entonces coordina tareas y equipos asumiendo el rol de lider.
1. Asegura de que el proyecto se entregue a tiempo y en el presupuesto establecido
2. Ejecuta el proyecto
3. Seguimiento y control: Monitorea el progreso, identifica desvios y aplica acciones correctivas
4. Planifica el proyecto definiendo objetivos, alcances y entregables
5. Coordinacion de recursos y comunicacion entre equipos y partes
6. Lidera el equipo
7. Gestion de riesgos y resolucion de problemas.
Los riesgos y las estrategias se planifican al comienzo del proyecto
###### Gestion de proyecto
Es importante que el proyecto puede ser exitoso si se pierde dinero pero cumple objetivos y en tiempo y forma.
#### Product manager - Estrategia del producto - NO es necesario en todos los proyectos

Prioridades:
1. Tener la vision y estrateguia del producto. Define la vision a largo plazo y la estrategia del producto alineada con los objetivos de empresa
2. Investiga el mercado, analisis de competidores y comprende las necesidades del cliente
3. Roadmap(caminio o evolucion en el tiempo) del producto. Desarrolla y mantiene el roadmap del producto priorizando las caracteristicas y funcionalidades
4. Coordinacion de stakeholders. Interactua con los interesados para asegurar que el producto se alinea con el mercado

#### Product owner
Son los que saben mas del producto y no tanto de la parte tecnica. 
>[!important]
>Maximiza el valor del producto entregado por el equipo de desarrollo.

Generalmente sirve para cuando hay una funcionalidad nueva, no tanto cuando se trata de mantenimiento. 
Cercano al equipo de desarrollo para cuidar los requisitos del producto. Intermediario entre desarrolladores y el cliente.
Como conoce bien los procesos de negocio es el interprete entre mi cliente y mi equipo de desarrollo.

1. Gestiona el backlog. Prioriza y gestiona el backlog asegurando que las historias de usuario (casos de uso) esten definidas y priorizadas. Cada user story tiene los requerimientos de mi producto obtenidos durante el analisis.
2. Aceptacion de entregable. Acepta o rechaza los resultados de trabajo entregados por el equipo de desarrollo, asegurando que cumplan criterios de aceptacion
3. Trabajo con el equipo de desarrollo. Trabajo estrechamente con el equipo de desarrollo para asegurarse de que entiendan los requisitos y se entreguen con las expectativas.
4. Criterios de aceptacion. Define y comunica criterios de aceptacion para garantizar que el producto cumpla con necesidades del usuario y los objetivos del negocio.
#### Scrum master
Es el encargado de asegurar que el proceso scrum se desarrolle correctamente.
1. Facilitador de scrum. Asegura que se sigan las practicas y valores de scrum, facilitando eventos como sprints, dayili stand-ups, sprint reviews y retrospectives
2. Coaching. Promueve la auto-organizacion y mejora continua dentro del equipo fomentando la colaboracion y el desarrollo de competencias
3. Eliminacion de obstaculos. Ayuda a identificar y eliminar impedimentos que puedan afectar al progreso de los desarrolladores
Mejora la eficiencia del equipo de desarrollo y asegura la adherencia a las practicas de scrum.

## Ciclo de vida de un proyecto
Tiene la idea de que sigue un proceso para llevar adelante a un proyecto. Es un conjunto de fases ordenadas e interconectadas para garantizar el exito de un proyecto. Este ciclo lo sigue el **Project manager**
![[Pasted image 20260601201244.png]]
Todo proyecto tiene como minimo 5 etapas (desde el punto de vista de un proyecto). Debe evaluar como llevar adelante estas faces.

>[!note]
>Cada KA de proyectos del PMI coincide con el ciclo de vida siguiente
### 1. Estructuración o Iniciación
Se define el proyecto a alto nivel, se identifican necesidades, objetivos y viabilidad.

### 2. Planificación
Se detallan los objetivos, alcance, recursos, tiempos, costos y riesgos.
Se elabora el plan del proyecto. 

Finalizada estas etapas se realiza el kickoff tecnico. Es donde se conocen los miembros del proyecto y expresan conformidad frente a la estructuracion y planificacion

>[!important]
> Estas 2 faces son **iterativas** ya que a medida que se avance el proyecto se vuelve para atras o se repiensa.
### 3. Ejecución
Se lleva a cabo el plan del proyecto para producir los entregables definidos. Se lleva a cabo segun la planificacion establecida.
### 4. Control y Seguimiento
Se monitorea el desempeño del proyecto, se comparan resultados con el plan y se toman acciones correctivas cuando es necesario. Si existen retrasos debe actualizar la planificacion y por ende se puede volver a dicha etapa
### 5. Cierre
Se finaliza formalmente el proyecto, se entregan los entregables y se documentan las lecciones aprendidas. Garantiza que lo generado en el proyecto quede como activo de la organización.
#### Ciclo continuo de mejora y aprendizaje
Proceso transversal que permite retroalimentar y optimizar la gestión del proyecto durante todo su ciclo de vida.

### Factores de exito de un proyecto - Triple limitacion
El exito del proyecto implica equilibrar estas variables. Estas variables al mismo tiempo deben ser negociadas con el cliente. Al mismo tiempo se modifican/limitan.
#### Alcance
El trabajo se realiza para cumplir los objetivos. Es el primero que se establece. Una vez establecido se ven las otras variables que se ven modificadas.
#### Tiempo
El plazo definido para completar el proyecto
#### Costo
El presupuesto aprobado para realizar el proyecto
#### Otras variables que se ven modificadas por estas 3 dimensiones
- Calidad
- Riesgos
- Capital intelectual
- Comunicaciones
- Recursos materiales
- Cambios de alcance
- Interesados
- Adquisiciones
Para el caso del agilismo el costo y el tiempo son **fijos**

## Fases del modelo

### Fases del modelo predictivo (PMBOK)

El modelo predictivo sigue un enfoque secuencial y estructurado, donde el control está presente durante toda la ejecución.

- **Iniciación:** Definición del proyecto y autorización para comenzar.
    
- **Planificación:** Definición del alcance, tiempo, costo y otros planes subsidiarios.
    
- **Ejecución:** Coordinación de personas y recursos para cumplir con el plan.
    
- **Control y Seguimiento:** Monitoreo constante del desempeño para identificar desviaciones y aplicar acciones correctivas.
    
- **Cierre:** Formalización de la aceptación del producto y finalización del proyecto.
    

### Fases del Agile Project Management (APM)

El modelo ágil se basa en ciclos iterativos, permitiendo la adaptación constante a los cambios.

- **Envision (Visión):** Establecimiento del propósito y visión del producto.
    
- **Speculate (Especulación):** Planificación basada en el conocimiento actual, creando la lista de características (_Feature List_) y el plan de versiones.
    
- **Explore (Exploración):** Desarrollo técnico y entrega de funcionalidades en cortos períodos de tiempo.
    
- **Adapt (Ajustes):** Revisión de los resultados obtenidos (características completadas) y ajuste de los planes para la siguiente iteración.
    
- **Close (Cierre):** Finalización y entrega del producto terminado.
    

### Comparativa de equivalencias

|**PMBOK (Predictivo)**|**APM (Ágil)**|
|---|---|
|Iniciación|Envision (Visión)|
|Planificación|Speculate (Especulación)|
|Ejecución|Explore (Exploración)|
|Control y Seguimiento|Adapt (Ajustes)|
|Cierre|Close (Cierre)|

## Modelo predictivo
### Estructuracion o iniciacion
- Identifica la envergadura el proyecto
- Identifiacar y consesuar el objetivo del negocio y el objetivo del proyecto
- Se lleva a cabo la tarea de elaboracion y correcion del **documento general del proyecto**. Tambien se generan los documentos del proyecto
#### Acta de proyecto - Documento general del proyecto
Es el documento de inicio formal del proyecto. Contiene:
- **Que**
- **Como**
- **Donde**
- **Cuando**
Describe objetivos de negocio y de proyecto. Tambien sirve como acuerdo inicial entre interesados sobre proposito y alcance de proyecto. Esto ultimo puede evolucionar en un contrato de proyecto.
##### Caracteristicas
- Describe el alcance, entregalbes, equipo de implementacion, roles y responsabilidades
- Detalla el plan a alto nivel del proyecto
- Plantea la estrategia de minimizacion de riesgos
- Informa costos generales estimados para el proyecto
- Describe supuestos y restricciones bajo la cual se planifico el proyecto

#### Notas TP
1. El titulo tiene que ser descriptivo de lo que haga el proyecto y empezar con un sustantivo. Defino el alcance del proyecto desde el titulo.
2. Debe ser lo mas amplio posible (no solo diseno, tambien implementacion, analisis, etc)
