**Objetivos de la asignatura**
1. Integrar conocimientos de primer año articularlos con problemas de cuarto nivel para abordar la gestion de sistemas de informacion desde una mirada sistemica. Mirada sistema implica entender como funciona cada parte del sistema, es decir, integrar aplicaciones, elegir frameworks mas alla de entender el producto. *Una cosa es entender el producto, otra cosa es como contextualizarlo y como hacerlo andar*, es decir, como se mete con la vida real mi producto
2. Comprender como administrar proyectos, servicios y recursos tecnologicos en organizaciones considerando procesos, personas, infraestructura y datos. Entender el producto o problema mas alla del codigo o la ejecucion del proyecto y verlo mas como distintos recursos a administrar.
3. Analizar la organizacion y su estrategia para alinear tecnologia y negocion mediante herramientas de arquitectura empresarial, modelado y evaluacion. Basicamente, alinear los recursos o estrategia para apuntar a los objetivos de la organizacion entendiendo el negocio. *Conocer el negocio para dar una solucion*
4. Diferenciar y aplicar las diferentes metodologias segun corresponda (predictivas y agiles) para planificar, ejecutar, monitorear y cerrar proyectos de sistemas de informacion.
5. Desarrollar criterio para la toma de decisiones sobre alcance, tiempo, costos, calidad, riesgos, adquisiciones, comunicaciones e interesados.
6. Fortalezer capacidades de **liderazgo**, trabajo en **equipo**, etica **profesional** y **comunicacion** efectiva para conducir iniciativas tecnologicas con impacto real. *Parte humana*
**Particion de la asignatura**
- Primer etapa - Aplicar conceptos de arquitectura empresarial - Que pasa si estoy dirigiendo un area IT y como se relaciona cross-organizacion. Busca integrar conceptos de cursada y tiene parcial. Para esto necesito entender la **arquitectura empresarial** y los aspectos de la empresa y asi elegir las herramientas para cada momento.
- Segunda etapa - entender y ver como gestionar los entregables cosa de manejarme con estrategia **cascada** o **predictivo** 
- Tercer etapa - Ver que pasa para este proyecto un modelo agil por sobre uno predictivo
