## ¿Que es y para que se usa?
Representa elementos (causas) de un sistema que pueden contribuir a un problema (efecto). Un diagrama ataca un problema. 
Es una herramienta para ayudara a los equipos de mejora coninua a tener una conceprion comun de un problema complejo. Recolecta datos y estudia procesos y situaciones.
>[!SUMMARY]
>Es utilizado para identificar las posibles causas de un problema específico. La naturaleza gráfica del diagrama permite que los grupos organicen grandes cantidades de información sobre el problema y determinar exactamente las posibles causas. Aumenta la probabilidad de identificar las causas principales
>

Cada espina es una causa y sus subcausas. Todas llevan a un problema. Entre todas las causas sumo 100%. En el ejemplo:
- Persona -> Personas individuales
- Organizacion -> Problemas de organizacion o politicas en general. Mas colectivo
- Proceso -> Problemas en el proceso o lo que hago
- Maquinas/Herramientas -> Son las herramientas y sus problemas con las que hago el proceso
### Problemas 6M
Son las principales causas a incluir en el diagrama
- Mano de obra
- Maquinaria
- Metodos
- Materiales -> Codigos datos dependencias
- Medio ambiente -> Contexto organizacional
- Medidas -> metricas
Generalmente el porcentaje de participacion en el problema de estas causas es subjetivo.
