
1. Defina con sus palabras qué es la ingeniería del software 
>	Es el enfoque sistemico o de nivel mas alto crear y mantener software mas alla del codigo como tal y enfocandose mas en la estructura
2. Su definición tiene alguna diferencia/discrepancia con la definición de la IEEE. Cuál 
>	 La diferencia recae en que mi definicion habla mas de la estructura del codigo y la definicion de metodos y cuantificaciones
3. Qué características tiene el software como producto 
> 	 Es mantenible y repensable. Muchos de los problemas que se encuentran a la hora de desarrollarlo son bastante comunes y repetitivos. Como tal sus consecuencias y causas tambien lo son. Enfocarse en dichos aspectos puede ayudar mucho al desarrollo.
4. Qué es el SWEBOK 
> 	 Es una guia que describe el conocimiento que esta generalmente aceptado en la ingenieria de software.
5. Nombre 3 de las Key Areas del SWEBOK 
> 	 - Ing en software practica profesional -> Se encarga de conocimientos, habilidades y actitudes de los ingenieros en software para practicar la ingenieria como una profesion
> 	 - Economia de la ingenieria en software -> Trata de tomar decisiones de software en un contexto empresarial. El existo de un producto de sw, servicio y solucion depende de una buena administracion
> 	 - Fundamentos matematicos -> Sopn tecnicas que puede representar y llevar adelante el razonamiento y el juecio de un ingeniero en software.

9. Qué es el diagrama de Espina de Pescado y para qué sirve 
>	 Representa elementos (causas) de un sistema que pueden contribuir a un problema (efecto). Un diagrama ataca un problema. 
	   Es una herramienta para ayudara a los equipos de mejora coninua a tener una conceprion comun de un problema complejo. Recolecta datos y estudia procesos y situaciones.
10. Describa los pasos a seguir para utilizarlo
> 	 Primero se debe obtener un problema que se puede atacar. Dicho problema tiene causas distribuidas en distintos areas. Cada causa debe formar una espina de pescado que a su vez, debe tener varias sub-causas que la componen. De esta forma se puede ver el problema de mas alto nivel.
