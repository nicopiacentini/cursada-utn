# Estrategias de Branching

Conjunto de reglas que definen cómo los desarrolladores interactúan con una base de código compartida al usar un sistema de control de versiones.

## Objetivos

- Mejorar la productividad garantizando coordinación entre desarrolladores
- Facilitar el desarrollo paralelo
- Organizar lanzamientos planificados y estructurados
- Trazar un camino claro desde el cambio hasta producción
- Mantener un código libre de errores donde los problemas se puedan corregir rápidamente sin interrumpir el flujo de trabajo

## Contexto

Las estrategias de branching son parte del [[Plan de gestion de configuración]] y se relacionan directamente con las prácticas de [[CI&CD and devOps#Integración Continua (CI)|CI/CD]].
